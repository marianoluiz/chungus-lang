## Dependency Management with pip-tools

This project uses pip-tools to manage pinned requirements files based on simple input files:

- Base app deps: [requirements.in](requirements.in)
- Dev/test deps: [requirements-dev.in](requirements-dev.in)
- Pinned/locked files (autogenerated):
  - [requirements.txt](requirements.txt)
  - [requirements-dev.txt](requirements-dev.txt)


### Compile locked requirements
- Compile production requirements from [requirements.in](requirements.in):
```sh
pip-compile requirements.in
```
- Compile dev requirements from [requirements-dev.in](requirements-dev.in):
```sh
pip-compile requirements-dev.in
```
This will generate/update [requirements.txt](requirements.txt) and [requirements-dev.txt](requirements-dev.txt) with pinned versions.

### Sync your environment
Use `pip-sync` to make your venv match the locked files exactly.

- For production-only:
```sh
pip-sync requirements.txt
```

- For development (includes prod + dev tools):
```sh
pip-sync requirements.txt requirements-dev.txt
```

### Add or update dependencies
- Add a runtime dependency: edit [requirements.in](requirements.in), then re-run:
```sh
pip-compile requirements.in && pip-sync requirements.txt
```

- Add a dev/testing tool: edit [requirements-dev.in](requirements-dev.in), then:
```sh
pip-compile requirements-dev.in && pip-sync requirements.txt requirements-dev.txt
```
